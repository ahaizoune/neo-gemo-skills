# Product Knowledge Base Governance

Use this reference when discovery work reveals a change to the GemForge platform picture.

## When To Update The Knowledge Base

Update `../gemo-foundation/references/product-feature-map.md` when a feature:

- adds a new platform capability
- materially changes a workflow stage
- changes upstream or downstream dependencies between features
- renames a user-facing or platform-facing concept that should remain stable across teams

## Update Rules

- update the knowledge base in the same body of work
- keep names aligned with current delivery artifacts
- reflect cross-feature dependencies, not only the local feature
- capture typical concerns and likely repo impact if they changed

## Closure Rule

Do not treat substantial product discovery as complete if the shared product picture is now stale.
