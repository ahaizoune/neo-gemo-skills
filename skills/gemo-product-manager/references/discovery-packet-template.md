# Discovery Packet Template

Use this reference when `gemo-product-manager` or `gemo-orchestrator` is shaping a feature before
formal grooming.

## Required Sections

1. Problem statement
2. Current pain or opportunity
3. Primary actors
4. Jobs to be done or desired outcomes
5. Current workflow versus desired workflow
6. Candidate feature map
7. Candidate sub-feature map
8. Key flows
9. Required states and edge cases
10. Explicit versus inferred behavior
11. Dependencies and hidden implications
12. MVP versus later phases
13. Success signals
14. Open product questions
15. Knowledge-base update needs
16. Likely repo and discipline impact

## Figma-Specific Checks

- Which flows are clearly represented in the design?
- Which states are missing?
- Which permissions or role differences are implied but not shown?
- Which backend or scheduling assumptions are hidden behind the UI?

## Knowledge Base Check

- Does this feature add or change a platform capability?
- Does the shared product feature map need an update before closure?
- Are we using names that stay consistent with the platform knowledge base?
