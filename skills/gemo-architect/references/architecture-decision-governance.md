# Architecture Decision Governance

Use this reference when the architect is making or revising meaningful technical decisions.

## Rules

- assign a decision ID when an option is chosen
- record the decision in `decisions.md`, not only in the architecture packet
- state whether the decision supersedes an older one
- make deferred decisions explicit instead of burying them in prose

## Decision Quality Bar

- the chosen option is unambiguous
- tradeoffs are explicit
- consequences are visible
- superseded decisions are traceable
